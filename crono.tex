\section{Montagem de plataforma para desenvolvimento do sistema}
\textbf{Status: Em andamento, no prazo.} \\
Dependências: nenhuma. \\
Objetivos:
\begin{listae}
	\item Definir que recursos serão utilizados no desenvolvimento.
\end{listae}
Resultados obtidos até o momento:
\begin{lista}
	\item A princípio, não será nececessário obter licenças para desenvolvimento ou manutenção do software.
	\begin{itemize}
		\item O sistema será desenvolvido com o Visual Studio Express 2013 for Web, que pode ser obtido gratuitamente no site da Microsoft.
		\item As bibliotecas gráficas especiais da DevExpress não serão mais empregadas. Em seu lugar, serão usados apenas objetos padrão da Microsoft.
	\end{itemize}
	\item O sistema usará o menor número de recursos possível, de forma a rodar na mais ampla variedade de plataformas.
	\begin{itemize}
		\item Serão realizados testes tanto na plataforma disponibilizada pela Gerdau Cosigua quanto em outras duas disponibilizadas pela Techsteel (ver descrição abaixo).
		\begin{enumerate}
			\item Plataforma de execução (cliente)
			\begin{itemize}
				\item Servidor SQLServer 2008 R2 Standard sobre Windows Server 2008
				\item Servidor Internet Information Server 7.0 sobre Windows sobre Windows Server 2008
				\item Admite-se a concessão de full-trust para o aplicativo
			\end{itemize}
			\item Plataforma de testes Web
			\begin{itemize}
				\item Servidor SQLServer 2008 R2 64 bits sobre Windows Server 2008
				\item Servidor Internet Information Server 7.5 sobre Windows sobre Windows Server 2008
				\item Admite-se apenas a concessão de medium-trust para o aplicativo
			\end{itemize}
			\item Plataforma de testes local
			\begin{itemize}
				\item SQL LocalDB e IIS Express sobre Windows 8.1
				\item Admite-se a concessão de full-trust para o aplicativo
			\end{itemize}			
		\end{enumerate}
		\item As bibliotecas gráficas especiais da DevExpress não serão mais empregadas. Em seu lugar, serão usados apenas objetos padrão da Microsoft.
		\item O cliente usará navegador Google Chrome, versão mais atualizada.
		\item O aplicativo desenvolvido pela VirtualWorks estava usando a versão 4.0 do ASP.NET. A versão atual é a 4.5. A princípio, deve ser possível usar esta última sem problemas. Isso deve ser testado pela versão preliminar do aplicativo.
		\item Aparentemente, o aplicativo desenvolvido pela VirtualWorks estava usando Google Charts. Se possível, vamos evitar mais essa dependência, e usar apenas ferramentas padrão do Visual Studio.
	\end{itemize}
\end{lista}
Entregáveis:
\begin{listae}
	\item Versão preliminar do aplicativo, funcional nas 3 plataformas listadas acima, exibindo as funcionalidades básicas de:
	\begin{enumerate}
		\item login e cadastramento de usuários
		\item navegação de telas
	\end{enumerate}
\end{listae}
Previsão para conclusão: 19/06/2015. \\
Informações e/ou ações necessárias pelo cliente:
\begin{listaef}
	\item Características da plataforma de execução. \color{PineGreen} \textbf{Fornecidas em e-mail datado de 12/06/2015.} \color{black}
	\item Liberação de acesso para instalação de software e gerenciamento do banco de dados. \color{Plum} \textbf{Solicitadas por e-mail datado de 15/06/2015.} \color{black}
\end{listaef}
